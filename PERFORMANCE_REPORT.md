# Performance Report

## 改善内容

- メモ一覧の Prisma `findMany` に `take`、URL limit、サーバー側 `orderBy` を追加し、初期表示で全件を取得しないようにしました。
- 一覧用の Prisma `select` を詳細/編集用と分離し、Todo プレビューは最大 5 件、共有ロールは 1 件だけ取得します。
- 長文メモは一覧レスポンスに渡す前にプレビュー文字数へ切り詰め、詳細表示時だけ個別 API から全文を取得する形にしました。
- Todo 横断一覧とモバイル Todo API に `take` と安定した `orderBy` を追加し、初期取得量を制限しました。
- Web のメモ一覧と Todo 一覧に「もっと見る」を追加し、必要な場合だけ取得上限を増やします。
- Web/Expo の検索・集計処理を小さな配列と memoized/deferred な計算に寄せ、入力時の再計算を抑えました。
- モバイルのメモ/Todo/カレンダー API 呼び出しに limit を渡し、アプリ状態に保持する初期データ量を減らしました。

## 既存機能への配慮

- 詳細画面、編集画面、Todo 操作、共有/公開/削除の権限チェックは既存のサーバーアクションと個別 API を維持しています。
- 一覧ではプレビューだけを使い、詳細を開いた時に全文と全 Todo を取得するため、編集対象のデータ欠落を避けています。
- 既存の認証・モバイル Bearer token・PostShare 権限チェックの構造は変更していません。

## 残課題

- 検索は現在、取得済みページ内では軽量化されていますが、PostgreSQL 側の全文検索インデックスは未導入です。大量データで全件横断検索が必要なら `tsvector` や trigram index の検討余地があります。
- 一覧の件数表示は取得済みデータベースの件数とページ内集計を併用しています。フィルターごとの完全な内訳が必要なら、集計専用クエリを追加できます。
- モバイルには「もっと見る」UI までは追加していません。API とクライアント関数は limit を受け取れるため、必要になれば一覧末尾の追加ロードに拡張できます。
